Login portal de identificação

O pedido veio de um terceiro, através do gateway Tyk, para tentar acessar um recurso OAuth. O gateway Tyk identificou a API a que pertence e redirecionou o cliente para o seu provedor de identidade de login (você controla este formulário).

Como o pedido que gerou esta página é um redirecionamento POST 307, os dados do POST de origem devem ser mantidos e você pode ver as variáveis abaixo.

  • Client ID: {{.ClientID}}
  • Redirect URL: {{.RedirectURI}}
  • Response Type: {{.ResponseType}}
